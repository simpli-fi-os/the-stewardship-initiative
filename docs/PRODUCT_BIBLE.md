## 1. Executive Summary

### What TSI Is

The Stewardship Initiative is a skills-and-services directory platform for high-trust communities, starting with churches. It makes invisible networks visible by connecting member needs with member gifts.

### What TSI Is NOT

TSI is not a Church Management System (ChMS). It does not handle giving, attendance, check-in, worship planning, or accounting. It does ONE thing: surfaces the hidden talent within a congregation and makes it findable.

This is a deliberate strategic choice. The ChMS market is crowded with 40+ competitors (Planning Center, Breeze, ChurchTrac, Tithely, ChMeetings). The "skills directory" niche within church software is underserved. Nobody is doing it well. That gap is our moat.

### The $1K/mo Thesis

Reach $1,000/month in recurring revenue with a lightweight, largely automated operation:

- **Conservative path**: 67 churches at $15/mo = $1,005/mo
- **Moderate path**: 34 churches at $29/mo = $986/mo
- **Premium path**: 20 churches at $49/mo = $980/mo

These are achievable within 6-12 months given that there are ~370,000 churches in the US. We need to convert 0.009% to 0.018% of the addressable market.

## 2. Market Analysis & Opportunity

### The Landscape

**US Church Market**:
- ~370,000 religious congregations in the United States (Hartford Institute)
- 332,000 are Protestant and other Christian churches
- ~70% of all churchgoers attend the largest 10% of congregations (250+ regular participants)
- The church management software market was valued at $970.9M in 2024, projected to hit $1.47B by 2033 (4.7% CAGR)
- 86% of churches now use some form of digital management tools

**The Disruption Moment**:
- 15,000 churches are projected to close in 2025 alone (Thom Rainer / LifeWay)
- 29% of Americans now identify as religiously unaffiliated
- Churches that survive are doubling down on community connection as their differentiator
- "Connection gap" is the #1 pastoral concern: 1 in 4 churchgoers feel disconnected (Barna Group)

**What this means for TSI**: Churches are desperate for tools that strengthen community bonds. The ones that survive the current contraction will be the ones that foster genuine member-to-member connection. TSI is exactly that tool.

### Competitive Pricing Analysis

| Competitor | What They Do | Monthly Price | Our Differentiator |
|-----------|-------------|---------------|-------------------|
| Instant Church Directory | Photo directory (contact info only) | $9.99-$14.99/mo | We surface SKILLS, not just names |
| Planning Center | Full ChMS (people, giving, services) | $0-$300/mo (per module) | We're not competing; we complement |
| ChurchTrac | Full ChMS | $9-$45/mo | We do one thing better than their directory tab |
| Breeze | Full ChMS | $72-$180/mo | Overkill for the directory use case |
| ChMeetings | Full ChMS | $0-$99/mo | We're lighter, faster, and purpose-built |

**Key insight**: Nobody is building a dedicated skills/services directory for churches. The existing players either do simple photo directories (names + faces) or full ChMS platforms where the "directory" is an afterthought buried in a settings tab. TSI owns the space between "who are our members" and "what can our members DO for each other."

## 5. Technical Architecture & Stack

### Current State (Alpha)

Static PHP pages, Tailwind CDN, vanilla JS, hardcoded demo data. This got us to MVP. Now we need a real stack.

### Target Stack: Next.js + Vercel + Supabase

| Layer | Technology | Why |
|-------|-----------|-----|
| **Framework** | Next.js 14+ (App Router) | SSR for SEO, API routes for backend, Vercel-native |
| **Hosting** | Vercel | Auto-deploy from GitHub, edge functions, great DX |
| **Database** | Supabase (PostgreSQL) | Row-level security, realtime, auth, free tier generous |
| **Auth** | Supabase Auth | Magic link (no passwords for church members to forget) |
| **Payments** | Stripe | Subscriptions, webhooks, customer portal |
| **CMS / Blog** | MDX files in repo (or Notion API) | Content lives in Git, auto-deploys with code |
| **Email** | Resend (or Loops) | Transactional + marketing, developer-friendly |
| **Analytics** | Plausible or PostHog | Privacy-respecting, no cookie banner needed |
| **Images** | Cloudinary or Vercel Image Optimization | Responsive, WebP, lazy loading |
| **DNS/Domain** | Vercel Domains + Cloudflare | SSL, CDN, subdomain routing for white-label |
| **Cron Jobs** | Vercel Cron (vercel.json) | Scheduled content generation, data sync |
| **AI Content** | Anthropic API (Claude) | SEO articles, listing enrichment |

### Database Schema (Supabase)

```sql
-- Organizations (churches)
CREATE TABLE organizations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  slug TEXT UNIQUE NOT NULL,
  description TEXT,
  logo_url TEXT,
  hero_image_url TEXT,
  website_url TEXT,
  primary_color TEXT DEFAULT '#022C22',
  accent_color TEXT DEFAULT '#FDB833',
  tier TEXT DEFAULT 'seed' CHECK (tier IN ('seed', 'growth', 'whitelabel')),
  stripe_customer_id TEXT,
  stripe_subscription_id TEXT,
  notion_api_key_encrypted TEXT, -- For orgs still using Notion
  notion_database_id TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Members (people in a directory)
CREATE TABLE members (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id UUID REFERENCES organizations(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  title TEXT,
  email TEXT,
  phone TEXT,
  website TEXT,
  photo_url TEXT,
  bio TEXT,
  description TEXT, -- What they can help with
  available BOOLEAN DEFAULT TRUE,
  featured BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Skills (many-to-many)
CREATE TABLE skills (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id UUID REFERENCES organizations(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  color TEXT DEFAULT '#6B7280',
  icon TEXT, -- Lucide icon name
  UNIQUE(org_id, name)
);

CREATE TABLE member_skills (
  member_id UUID REFERENCES members(id) ON DELETE CASCADE,
  skill_id UUID REFERENCES skills(id) ON DELETE CASCADE,
  PRIMARY KEY (member_id, skill_id)
);

-- SEO / Blog content
CREATE TABLE articles (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  slug TEXT UNIQUE NOT NULL,
  title TEXT NOT NULL,
  description TEXT,
  content TEXT NOT NULL,
  author TEXT DEFAULT 'The Stewardship Initiative',
  published BOOLEAN DEFAULT FALSE,
  published_at TIMESTAMPTZ,
  og_image_url TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Analytics events
CREATE TABLE directory_views (
  id BIGSERIAL PRIMARY KEY,
  org_id UUID REFERENCES organizations(id),
  member_id UUID REFERENCES members(id),
  event_type TEXT NOT NULL, -- 'view', 'search', 'contact_click', 'filter'
  metadata JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Row-Level Security (Supabase)

```sql
-- Members can only view profiles in their organization
CREATE POLICY "Members see own org" ON members
  FOR SELECT USING (
    org_id IN (SELECT org_id FROM org_members WHERE user_id = auth.uid())
  );

-- Org admins can manage their own org's data
CREATE POLICY "Admins manage own org" ON members
  FOR ALL USING (
    org_id IN (SELECT org_id FROM org_admins WHERE user_id = auth.uid())
  );
```

## 6. Automation Pipeline: The Autonomous Engine

This is the core of the "make money while you sleep" strategy. Three automated systems running on Vercel Cron:

### System 1: Content Engine (SEO Articles)

**Schedule**: Twice per week (Monday & Thursday, 6am CT)

**Flow**:
```
Cron triggers /api/cron/generate-article
         │
         ▼
Claude API generates article from topic queue
(system prompt includes TSI voice, SEO keywords, CTA templates)
         │
         ▼
Article saved to Supabase (published: false)
         │
         ▼
Admin reviews in dashboard (or auto-publishes after N successful generations)
         │
         ▼
Published article renders as static page via Next.js ISR
         │
         ▼
Sitemap auto-updates, Google Search Console pinged
```

**Topic Categories** (rotating):
1. "How to build a church skills directory" (bottom-funnel, direct intent)
2. "Stewardship in the modern church" (theological/thought leadership)
3. "Community connection strategies for churches" (mid-funnel)
4. "Church volunteer management best practices" (adjacent keyword)
5. "[Specific skill] in the church" (long-tail: "finding an accountant in your church")
6. "Church member engagement ideas" (top-funnel, high volume)

**Target**: 8 articles/month, 1,200-2,000 words each, targeting long-tail keywords with low competition and clear church-admin intent.

### System 2: Listing Enrichment (Directory Data)

**Schedule**: Daily at 3am CT

**Flow for new member submissions**:
```
Member submits profile via form (Notion Form or Supabase form)
         │
         ▼
Cron job picks up new/incomplete profiles
         │
         ▼
If website URL provided:
  → Fetch website, extract business description, services offered
  → Claude API summarizes into TSI format
         │
         ▼
If photo missing:
  → Flag for admin to request
  → Send automated email: "Complete your profile with a photo"
         │
         ▼
If skills/categories incomplete:
  → Claude API suggests categories based on title/description
  → Admin reviews or auto-applies high-confidence suggestions
         │
         ▼
Updated profile saved to database
```

**Important**: We are NOT scraping or stealing anyone's data. Every listing is opt-in. Members or church admins submit information voluntarily. The enrichment system simply improves the quality of what they submit by pulling from their OWN websites and public business information.

### System 3: Notion Sync (For Notion-Powered Orgs)

**Schedule**: Every 30 minutes

**Flow**:
```
Cron triggers /api/cron/sync-notion
         │
         ▼
For each org with notion_api_key:
  → Query Notion database for updated_at > last_sync
  → Diff with Supabase records
  → Upsert changes
  → Update last_sync timestamp
         │
         ▼
New/updated profiles appear in directory within 30 minutes
```

### System 4: GitHub Auto-Push (Content & Config)

**Flow** (for content stored in MDX):
```
Article generated → saved as MDX file in /content/blog/
         │
         ▼
Git commit via Vercel API or GitHub API
  (commit message: "content: auto-generated article - {title}")
         │
         ▼
Push to main branch
         │
         ▼
Vercel auto-deploys
         │
         ▼
New article live on site
```

For database-stored content (Supabase approach), this step is unnecessary since content is fetched at request time with ISR caching.

**Recommendation**: Use Supabase for articles (not MDX in git). Simpler, no git commits needed, admin can edit in a dashboard, and ISR handles caching. Reserve git pushes for code changes only.

#### Typography

| Role | Font | Weight | Usage |
|------|------|--------|-------|
| Display / Headlines | **Playfair Display** | 700 | Hero headlines, section titles, emotional moments |
| UI / Body | **Inter** | 300-700 | Body text, UI elements, navigation, buttons |

**Note on Inter**: While Inter is often considered generic in the frontend-design skill, it is the correct choice for TSI's body text because it pairs exceptionally with Playfair Display (serif + sans-serif contrast), reads cleanly at small sizes in the directory cards, and conveys the modern, accessible, approachable quality the brand needs. The "unforgettable" design moment in TSI is not the body font; it is the constellation animation, the Garden of Eden palette, and the heartbeat animation on "Heart."

**Hierarchy**:
```
Display (Playfair):  text-6xl / text-5xl  → Hero headlines
Section (Playfair):  text-4xl / text-3xl  → Section headings
Subhead (Inter):     text-2xl / text-xl   → Card titles, step titles
Body (Inter):        text-base / text-lg  → Paragraph text
Caption (Inter):     text-sm              → Labels, metadata, pills
```

```

Members browse, use the directory, tell other church friends

         ▼
```

## Appendix B: TSI Voice Guide

### Writing Rules (Non-Negotiable)

1. **Never use em dashes.** Use commas, periods, or parentheses instead.
2. **Never use "leverage," "harness," "utilize," "synergy," or "paradigm."** Say the actual thing.
3. **Never start with "In today's world" or "In this article."** Start with the point.
4. **Scripture references should feel natural.** Quote when it serves the reader, not to prove you're churchy.
5. **Stats should be specific.** "1 in 4 churchgoers feel disconnected (Barna Group)" beats "Many people feel disconnected."
6. **Short paragraphs.** 2-3 sentences max. White space is your friend.
7. **Active voice always.** "Your congregation's talent is going unseen" not "Talent is being left unseen by your congregation."

### Voice Examples

| Context | Wrong | Right |
|---------|-------|-------|
| Hero headline | "A Comprehensive Community Engagement Platform" | "Your congregation's greatest resource is already here." |
| CTA button | "Submit" | "Access The Village Directory" |
| Feature description | "Our platform leverages Notion's API to seamlessly integrate directory management capabilities" | "You manage a simple Notion database. We turn it into a beautiful, searchable directory." |
| Blog intro | "In today's rapidly changing church landscape, it's more important than ever to..." | "76% of people want to volunteer their skills. They just don't know where they're needed." |
| Error state | "An error occurred" | "We couldn't load the directory right now. Try refreshing, or let us know at help@thestewardshipinitiative.org." |
